# AAAI 审稿评分校准

> 基于 50 篇 AAAI 2023-2026 获奖论文的定量基准数据，帮助校准审稿评分。
> 核心逻辑：不是"你的论文必须达到获奖论文的每一项数值"，而是"偏离均值太远时，reviewer 需要额外理由来说服自己"。

---

## 一、AAAI 投稿 → 接收的基本数据

| 指标 | 数值 |
|------|------|
| AAAI 历年投稿量 | ~9,000-12,000 篇 |
| AAAI 接收率 | ~20-25% |
| 获奖论文/接收论文 | ~50/~2000 ≈ 2.5% |
| 每篇论文审稿人数 | 3-4 位 reviewer |

**含义**：模拟审稿时，默认 baseline 是 **Borderline** ——大多数投稿最终被拒。需要正面证据来抬高评分。

## 二、基于获奖论文的结构校准

### 2.1 Abstract

| 指标 | 获奖论文基准 [N=50] | 偏离时的审稿信号 |
|------|-------------------|----------------|
| 词数 | 均值 160，IQR [133, 168] | <100 词：信息密度可能不足；>220 词：可能冗长 |
| "we" 使用 | 均值 2.4，80% 在 1-3 次 | 0 次：可能过于被动；>5 次：可能过度自我宣传 |
| 方法引出动词 | 48% 使用 we propose/introduce | 0：方法贡献不突出 |
| "however" | 34% 使用 | 作为标准转折，不用也可 |
| 记忆点 | 48% 有明显记忆点 | 无记忆点：reviewer 可能记不住这篇论文 |
| 五步弧线 | 理论型走"问题→答案"变体 | 弧线不完整：reviewer 需要翻正文才能理解 |

### 2.2 Title

| 指标 | 获奖论文基准 [N=50] | 偏离时的审稿信号 |
|------|-------------------|----------------|
| NAME:Subtitle 模式 | 58% | 标题无记忆点：可记忆度偏低 |
| Question 型 | 4%（罕见） | 用问号标题风险较高——需要正文确实给出了答案 |
| 长度 | 8-16 词 | 过长（>20 词）：不够聚焦 |

### 2.3 Introduction

| 指标 | 获奖论文基准（定性） | 偏离时的审稿信号 |
|------|-------------------|----------------|
| 痛点量化 | 每个痛点带具体代价 | 泛泛的 "existing methods are limited" → 痛点不成立 |
| 三重一致 | 痛点↔创新↔贡献一一对齐 | 不对齐 → 贡献组织混乱 |
| Teaser Figure | 多数在第一页有 Fig. 1 | 无 teaser → reviewer 需要更多精力理解方法 |
| Contributions 条数 | 3-4 条 | 1-2 条：贡献可能太少；>5 条：贡献可能被稀释 |

### 2.4 Experiments

| 指标 | 获奖论文基准（定性） | 偏离时的审稿信号 |
|------|-------------------|----------------|
| Ablation | 每个 module → 消融行 | 消融缺位 → 贡献的真实性存疑 |
| std/p-value | 大多数获奖论文至少报告 std | 无 variability → 可重复性存疑 |
| Baseline 数量 | ≥ 5（含经典+最新） | <5：baseline 覆盖不足 |
| 表格格式 | booktabs + 最优粗体 + 数字右对齐 | 格式混乱 → 不专业 |

## 三、评分边界定义

### Strong Accept（Top 5%）
- Q1（贡献）：Exceptional——贡献是"一块硬东西"，有清晰的 insight
- Q2（实验）：Flawless——消融完整、统计检验到位、baseline 公平
- Q3（写作）：清晰、愉悦、跨 track 可读
- Q4（格式）：完全合规
- 在获奖论文中也属于前 10%

### Accept（Top 20%）
- Q1：Solid contribution——有实质贡献，可能有 minor issues
- Q2：Well-executed——实验扎实，小的遗漏可接受
- Q3-Q4：基本清晰、合规
- 类似获奖论文中的中等水平

### Weak Accept（Top 35%）
- Q1：Good contribution but...——贡献有亮点但有局限性
- Q2-Q3：实验有 gap 或写作有模糊之处
- Reviewer 倾向于 accept 但希望看到 rebuttal 中的澄清

### Borderline
- Q1 或 Q2 有 major concerns
- 贡献不够突出 OR 实验有显著缺口 OR 写作严重影响理解
- 经常是 "interesting idea but insufficient evidence"

### Weak Reject
- Q1 或 Q2 有 critical issues
- 贡献 incremental 或实验不可信
- Reviewer 不认为 rebuttal 能解决核心问题

### Reject
- Q1：贡献缺失——"so what?" 没有答案
- OR Q2：实验有致命缺陷（数字矛盾、cherry-pick 明显、baseline 完全不公平）
- OR Q4：存在明确格式 `ERROR` 或双盲泄露（具体程序后果不得由本 skill 臆测）

## 四、按论文类型的评分校准

### 理论型（12/50 获奖论文）

| Score 提升因素 | Score 降低因素 |
|---------------|---------------|
| 定理陈述独立可读，不依赖周围文字 | 假设隐式嵌入段落中，未显式列出 |
| 证明直觉清晰传达（不展开所有 technical lemma） | 声称 "can be proved" 但未给证明草图 |
| 理论结果有实际 implications | 纯理论但没说 "so what" |
| 轻量实验作为概念验证即可，不强求大规模 benchmark | 实验全无——连 toy domain 验证都没有 |

### 模型方法型（23/50 获奖论文）

| Score 提升因素 | Score 降低因素 |
|---------------|---------------|
| 设计有 insight（"我们观察到 X，所以设计 Y"） | 像几个现成模块的组合（"我们把 A+B+C 拼在一起"）|
| SOTA 有 meaningful margin（>1-2%） | 多个 benchmark 上只 marginal improvement |
| 消融完整，每个 module 有对应消融行 | 消融缺失或敷衍 |
| 效率分析（params/FLOPs/inference time） | 只报 accuracy 不报 cost |

### 基准资源型（4/50 获奖论文）

| Score 提升因素 | Score 降低因素 |
|---------------|---------------|
| 社区长期价值清晰（"这篇论文会被广泛 cite 作为 benchmark"） | 数据集只有一次性使用价值 |
| 数据质量论证充分（inter-annotator agreement + 质控流程） | 数据采集流程模糊 |
| baseline 全景覆盖不同方法论范式 | baseline 少且浅 |
| 通过规模揭示新发现/洞见 | 只有规模没有洞见 |

### 应用驱动型（11/50 获奖论文）

| Score 提升因素 | Score 降低因素 |
|---------------|---------------|
| 真实问题的重要性和紧迫性让非领域专家也能感受 | 问题重要性只对领域专家有意义 |
| 每个 AI 设计决策有清晰的领域动机 | AI 方法和领域脱节——"我们用了一个标准模型" |
| 领域指标 + 案例分析 | 只用标准 ML benchmark |
| 诚实的局限性讨论（真实场景中的 failure mode） | 画饼——声称 impact 但无实际验证 |

## 五、使用时的重要提醒

1. **校准数据是参考，不是硬标准**：获奖论文的 abstract 均值 160 词，不代表 120 词就不能接受
2. **一篇论文可以偏离多个指标但仍然很强**——评分是 holisitic，不是逐项打分相加
3. **类型很重要**：不能拿模型方法型的实验标准去要求理论型论文
4. **AAAI 审稿方差较大**：3-4 个 reviewer 对同一篇论文的评分可能有显著差异——本模拟器给出的是"大概率"评估
