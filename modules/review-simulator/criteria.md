# AAAI 模拟审稿七维评价标准

> 使用本文件判断维度证据；使用 `scoring-rubric.md` 赋分和聚合。七维科学质量、Assessment Confidence、格式/匿名合规必须分别报告。

## 使用规则

1. 先确定论文主类型：理论型、模型方法型、基准资源型或应用驱动型。
2. 每个维度记录 `status / score / evidence / concern`，不得只给印象分。
3. 维度分使用 0–6、0.5 步进；`4` 表示总体达到可接受水平，不表示完美。
4. 用户没有提供材料时标 `UNAVAILABLE_TO_REVIEWER`；只有确认论文自身遗漏时才标 `MISSING_IN_PAPER`。
5. 同一问题只指定一个主扣分维度，避免跨维重复扣分。

## D1 Significance：问题重要性与价值

**核心问题**：论文解决的问题是否值得 AAAI 受众关注，成功解决后会产生明确的学术或现实价值？

**4 分锚点**：问题的重要性被具体说明，目标受众和潜在收益清楚，主要价值不依赖营销式表述。

| 论文类型 | 重点证据 |
|---|---|
| 理论型 | 回答了有意义的开放问题、改进了关键界限，或澄清了重要理论性质 |
| 模型方法型 | 解决了真实存在且有影响的技术瓶颈，而非只追求微小榜单变化 |
| 基准资源型 | 填补了社区资源、评估协议或数据覆盖的实质空白 |
| 应用驱动型 | 问题在目标领域重要，AI 方案的收益与受影响对象被具体说明 |

低分不能只因为题目“小众”；应指出价值链中缺失的具体环节。

## D2 Novelty：原创性与实质贡献

**核心问题**：核心贡献是否清楚、非平凡，并与最相关工作形成可核验的实质差异？

**4 分锚点**：贡献可用一句话复述；与 closest prior work 的差异具体、可信，并带来明确的新能力、理解或资源价值。

检查：

- 贡献不是仅重新命名、调参或无洞见的模块拼接；
- 原创性主张与 Related Work、方法和实验相互一致；
- 每项贡献都能追踪到正文机制和证据；
- “first”“novel”等措辞没有替代实质比较。

`CORE_NOVELTY_UNESTABLISHED` 只能在最相关工作和核心差异均有定位证据时触发，不能因 reviewer 尚未检索完整文献就触发。

## D3 Soundness：技术正确性与论证深度

**核心问题**：方法、假设、推导、证明、数据构建或应用设计是否逻辑成立，中心结论是否超出证据？

**4 分锚点**：核心技术链条正确且足以复核；假设、边界和局限性清楚；剩余问题不推翻主要结论。

| 论文类型 | 重点证据 |
|---|---|
| 理论型 | 定理条件、证明步骤、边界情况和结论范围 |
| 模型方法型 | 设计动机、算法定义、训练/推理逻辑和复杂度 |
| 基准资源型 | 采集、标注、质控、泄漏防护和评估协议 |
| 应用驱动型 | 问题建模、领域假设、安全边界和部署条件 |

只有能定位并解释为什么主要结论整体坍塌时，才触发 `FATAL_VALIDITY`。

## D4 Evidence：证据与评估可信度

**核心问题**：论文实际提出的每个中心 claim，是否由适当、公平、充分且可解释的证据支持？

**4 分锚点**：主要 claim 有对应证据；比较协议公平；关键不确定性得到合理处理；结果解释不超出数据或证明范围。

| 论文类型 | 重点证据 |
|---|---|
| 理论型 | 完整证明或可访问的证明链、必要实例、反例/边界讨论、实际含义 |
| 模型方法型 | 强相关 baseline、消融、稳健性/效率分析和任务所需的统计证据 |
| 基准资源型 | 数据质量、代表性、baseline 全景、协议可靠性和泄漏分析 |
| 应用驱动型 | 领域指标、现实约束、案例/专家验证、失败模式和外部有效性 |

不要机械要求所有论文都有 p-value、固定数量 baseline 或模型论文式 SOTA。先判断 claim 和评价协议是否确实需要这些证据。

`DECISIVE_EVIDENCE_MISSING` 只适用于论文自身确实没有中心 claim 所需证据；附件当前不可访问应标为 `UNAVAILABLE_TO_REVIEWER`。

## D5 Clarity：表达、组织与可理解性

**核心问题**：目标领域之外的合格 AAAI reviewer 能否准确理解问题、方法、证据和结论？

**4 分锚点**：论文整体清楚，关键术语和符号有定义，图表和论证链可跟随；局部表达问题不妨碍评价贡献。

检查：

- 摘要准确概括问题、方法和主要结果；
- 贡献、方法组件和证据可相互追踪；
- 公式、符号和缩写一致；
- 图表及 caption 足以支持正文论证；
- 写作问题与技术问题分开记录。

无 teaser、无方法缩写或标题不是 `NAME: Subtitle` 不构成自动扣分。

## D6 Related Work：文献覆盖与工作定位

**核心问题**：关键相关工作是否被完整、公平地讨论，论文是否准确说明自身位置？

**4 分锚点**：覆盖决定原创性判断的主要工作；比较公平；差异表述具体；不存在已确认的关键遗漏。

不得使用固定自引比例、引用数量或 Related Work 页数阈值。当前未做外部检索时，明确限制条件并降低 Confidence，而不是断言“完整覆盖”。

## D7 Reproducibility：可复核性与透明度

**核心问题**：读者是否获得足够信息来复核主要结论，并理解实现、数据、证明、质控和局限性？

**4 分锚点**：提供与论文类型相称的必要细节；关键选择和不确定性透明；官方 Reproducibility Checklist 的适用项有依据地回答。

| 论文类型 | 重点证据 |
|---|---|
| 理论型 | 假设、完整证明位置、推导细节、验证代码或实例（若适用） |
| 模型方法型 | 数据划分、超参数、随机性、计算资源、实现和评估协议 |
| 基准资源型 | 数据来源、许可、标注规范、质控、版本和访问方式 |
| 应用驱动型 | 领域数据处理、部署环境、评估者信息、伦理和限制 |

“代码未公开”本身不自动等于 0 分；评价论文是否提供了复核其 claim 所需的信息，并单独说明资源访问限制。

## 不进入科学加权分的状态

以下项目单独输出，不放入 D1–D7：

- Author Kit 格式与 `.tex` 合规；
- 匿名性和身份泄露风险；
- 具体 event 的页数、supplement、checklist 或 track policy；
- 未解决的伦理/安全审查问题；
- 当前输入材料不完整。

格式状态以 `rules/aaai27-format-rules.json` 和 `scripts/aaai27_check.py` 为准。不得把未确认的程序后果写成科学分扣分或统一 desk reject。

## Evidence Ledger 模板

```text
| Dimension | Status | Score | Evidence / Location | Main Concern |
|-----------|--------|------:|---------------------|--------------|
| Significance | ASSESSED | 4.0 | Abstract; §1 ¶2 | Value outside task X needs clarification |
| Novelty | ASSESSED | 3.5 | §1 contributions; §2 | Closest baseline Y lacks mechanism-level comparison |
| Soundness | UNAVAILABLE_TO_REVIEWER | N/A | Appendix not provided | Cannot verify Lemma 2 |
```

判定图表不自包含时，修复建议见 `modules/caption-writing.md`。
