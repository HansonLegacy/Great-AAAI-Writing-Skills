# AAAI 图表设计规范

> 图表是 reviewer 第一印象的核心组成部分。一张好的 teaser 图 = 半篇论文的说服力。
> 所有规则对齐 AAAI 2027 Author Kit。

---

## 一、AAAI 硬性约束

| 约束 | 说明 |
|------|------|
| **图片格式** | 仅 `.jpg`, `.png`, `.pdf`。禁用 `.gif`, `.eps`, `.ps` |
| **分辨率** | 位图 ≥ 300 dpi |
| **裁剪** | 必须 LaTeX 外完成。禁用 `clip=true`, `trim`, `viewport` |
| **组合图** | 禁用 `minipage` 组合多图——用单一 PDF 包含所有子图 |
| **pgfplots** | 不可在正文中实时编译——预导出为 PDF 再 `\includegraphics` |
| **浮动体位置** | 优先 `[t]`（页顶）、次选 `[b]`（页底）；不得集中堆放文末 |
| **跨栏图** | 用 `figure*` 环境 |
| **图注位置** | 在图**下方**，10pt Roman，不加粗不斜体 |
| **颜色** | WCAG 2.0 对比度 ≥ 4.5:1；灰度打印必须可读；正文不得用颜色命令 |

## 二、Teaser Figure 设计原则

### 2.1 三要素

1. **问题展示**：左边/上方——现有方法的问题是什么？（可视化）
2. **方法展示**：右边/下方——我们的方法如何解决？（可视化）
3. **关键标注**：用文字标注核心差异（不必拘泥，用少量文字说明）

### 2.2 三种类型

| 类型 | 布局 | 适用 | 获奖示例 |
|------|------|------|---------|
| **对比型** | 左：旧方法的问题 → 右：新方法的解决方案 | 你需要直观展示 "why ours is better" | LENS, Binary-Gaussian |
| **架构型** | 数据流从左到右、模块上下堆叠 | 你需要展示 pipeline 的结构 | LLM2CLIP, JoLT |
| **概念型** | 中心概念 + 环绕解释 | 你的核心贡献是一个抽象概念 | Every-Bit-Helps（两个场景） |

### 2.3 设计清单

- [ ] 第一页可见（用 `[t]` 放置，不沉到页底）
- [ ] 非领域专家也能大致看懂（颜色+文字标注引导）
- [ ] 无机构 logo/名称（匿名投稿）
- [ ] 分辨率 ≥ 300 dpi
- [ ] 灰度打印可读
- [ ] 图注自包含——reviewer 不读正文也能大致理解

## 三、Pipeline / Architecture Figure 设计

### 3.1 数据流方向

- 左→右（最自然）或 上→下
- 用**箭头**明确数据流方向
- 不同阶段用不同颜色/框线区分

### 3.2 关键标注

- **张量形状**：`B×C×H×W` 等关键维度标注在模块连接处
- **Loss 函数**：标出 loss 的计算位置
- **训练/推理**：用不同线型区分训练路径和推理路径（虚线 vs 实线）
- **模块名**：与 Method 的 subsection 名一致

### 3.3 常见错误

| 错误 | 问题 | 正确做法 |
|------|------|---------|
| 文字太小看不清 | AAAI 双栏下，图内文字容易被压缩到不可读 | 导出时确保图内文字 ≥ 7pt |
| 颜色太多 | 看起来混乱 | 2-4 种主色 + 灰度即可 |
| 无张量形状 | Reviewer 不知道数据维度 | 关键连接处标形状 |
| 模块边界不清 | 不知道什么是模块 A 什么是模块 B | 用不同背景色/框线区分 |

## 四、表格设计

### 4.1 AAAI 表格核心规则

- **包**：推荐 `booktabs`（`\toprule`, `\midrule`, `\bottomrule`）
- **字号**：10pt 正常；9pt 必要时（**不得更小**）
- **禁用**：`\resizebox`（绝对禁止）；`\tiny`
- **允许**：`\setlength{\tabcolsep}` 压缩列间距
- **表注**：在表**下方**

### 4.2 设计原则

1. **一瞥可读**：reviewer 不读正文也能通过 caption + 表头 + 粗体标注理解表格
2. **数字右对齐**、**文字左对齐**
3. **最优值粗体**（`\textbf{85.7}`）、**次优值下划线**（`\underline{84.1}`）
4. **分组**：用 `\midrule` 分隔逻辑组
5. **Δ 列**：消融表面向 reviewer——"Δ from full" 省去 reviewer 心算
6. **信息密度**：6-12 行 × 4-8 列为宜；超出考虑分表

### 4.3 跨栏表

当单栏容纳不下时：
1. 先尝试压缩列间距（`\setlength{\tabcolsep}{Xpt}`）
2. 仍不行 → 跨双栏 `table*`
3. 仍不行 → 拆为两表

## 五、颜色可读性

### 5.1 WCAG 2.0 要求

- 对比度 ≥ 4.5:1（正常文字）
- 文字颜色与背景的对比度
- **灰度打印测试**：将 PDF 转为灰度，所有图中的信息仍可区分

### 5.2 推荐配色

- **主色 2-3 种** + 灰度辅助
- 避免红绿纯色组合（色盲不可读）
- 推荐：蓝-橙 / 蓝-红 / 紫-黄

### 5.3 颜色使用位置

| 允许 | 禁止 |
|------|------|
| 图中使用颜色 | 正文文字使用颜色 |
| 表中少量文字颜色（标注最优） | 标题/节标题使用颜色 |
| 单词级别的强调 | 段落级别的着色 |

- [ ] 所有图 `.jpg` / `.png` / `.pdf` 三选一？
- [ ] 位图 ≥ 300 dpi？
- [ ] 裁剪在 LaTeX 外完成，无 `clip=true` / `trim` / `viewport`？
- [ ] 无 `minipage` 组合多图？
- [ ] 无 `pgfplots` 实时编译？
- [ ] 浮动体优先 `[t]` / `[b]`？
- [ ] 图注/表注在下方，10pt Roman？
- [ ] 表使用 `booktabs`，无 `\resizebox`，无 `\tiny`？
- [ ] 灰度打印可读？
- [ ] 正文无颜色命令？
- [ ] Teaser 图在第一页可见？
- [ ] 图中文字 ≥ 7pt（导出时确认）？
